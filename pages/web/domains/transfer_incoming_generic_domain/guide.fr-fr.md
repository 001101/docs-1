**Dernière mise à jour le 15/12/2017**

## Objectif

Le transfert d'un nom de domaine permet de changer le bureau d'enregistrement de ce dernier. Ce transfert est possible vers OVH et peut durer entre un et dix jours.

**Apprenez comment réaliser le transfert d'un nom de domaine générique vers OVH.**

## Prérequis
- Disposer d'un nom de domaine générique (avec une extension comme .com, .net, .info, etc.) déposé dans un autre bureau d'enregistrement.
- Être habilité à demander le transfert du nom de domaine. Le propriétaire et/ou les personnes l'administrant doivent être informés de la demande du transfert.
- Le domaine doit avoir été créé il y a plus de 61 jours.
- Le domaine ne doit pas avoir été transféré ou avoir changé de propriétaire durant les 61 derniers jours.
- Le nom de domaine doit être déverrouillé.
- Vous devez être en possession du code de transfert ou avoir la possibilité de le récupérer.
- Les adresses e-mail des contacts renseignées dans la base *Whois* doivent être valides.

## En pratique
### Étape 1 : vérifier les informations liées au domaine

Votre nom de domaine est actuellement déposé dans un bureau d'enregistrement. Vous souhaitez changer ce dernier et faire d'OVH votre nouveau bureau d'enregistrement ?

Cette action est possible via une procédure de transfert. Celle-ci implique plusieurs entités : OVH, votre bureau d'enregistrement actuel, ainsi que d'autres parties. Durant l'application de cette procédure, plusieurs validations seront nécessaires. **Il est donc important de vérifier si les informations liées au nom de domaine sont à jour.**

Pour ce faire, vous pouvez utiliser notre outil *Whois* accessible depuis le lien : [https://www.ovh.com/cgi-bin/whois.pl](https://www.ovh.com/cgi-bin/whois.pl){.external}. Le résultat obtenu affichera plusieurs informations.

- **Si elles sont correctes** : déverrouillez votre domaine auprès de votre bureau d'enregistrement actuel.

- **Si elles sont incorrectes ou ne sont pas visibles** : rapprochez-vous de votre bureau d'enregistrement actuel pour les vérifier et/ou les modifier.

> [!primary]
>
> Si vous ne connaissez pas votre bureau d'enregistrement actuel, les lignes "Registrar", qui apparaissent dans le résultat de l'[outil](https://www.ovh.com/cgi-bin/whois.pl){.external} *Whois*, peuvent vous donner une indication sur son identité.
>

Voici un tableau résumant les différentes étapes d'un transfert.

|Étapes|Description|Qui est impliqué ?|Où ?|Délais|
|---|---|---|---|---|
|1|Déverrouiller votre domaine|L'administrateur du domaine, sous l'accord du propriétaire|Au bureau d'enregistrement actuel|Selon votre action|
|2|Récupérer le code de transfert|L'administrateur du domaine, sous l'accord du propriétaire|Au bureau d'enregistrement actuel|Selon votre action|
|3|Demander le transfert du domaine|Quiconque, en possession du code de transfert, et toujours sous l'accord du propriétaire|Au nouveau bureau d'enregistrement (OVH par exemple)|Selon votre action|
|4|Première étape de validation du transfert|Le propriétaire du domaine ainsi que l'administrateur renseigné chez le nouveau bureau d'enregistrement|Via un e-mail reçu|Cinq jours maximum|
|5|Deuxième étape de validation du transfert|Le bureau d'enregistrement actuel|Via une demande de la part de l'organisme qui gère l'extension de votre domaine|Cinq jours maximum|

### Étape 2 : déverrouiller le domaine

Une fois les informations vérifiées, il est nécessaire de déverrouiller votre nom de domaine. Cette manipulation se réalise uniquement via votre bureau d'enregistrement actuel. Nous vous invitons à prendre contact avec ce dernier pour connaître leur procédure.

Une fois votre nom de domaine déverrouillé, votre bureau d'enregistrement doit vous communiquer le code de transfert qui lui est associé. Différentes appellations peuvent être utilisées pour désigner ce code : **code de transfert**, **AuthCode**, **AuthInfo** ou encore **code EPP**.

Notez qu'OVH n'est pas le bureau d'enregistrement actuel de votre nom de domaine. Nous ne sommes donc pas en mesure de déverrouiller celui-ci ni de récupérer son code de transfert.

> [!warning]
>
> À compter du déverrouillage de votre nom de domaine, vous disposez de sept jours pour initier son transfert chez OVH. Au-delà de ce délai, celui-ci se verrouille automatiquement si aucune demande de changement de bureau d'enregistrement n'a été faite.
>

### Étape 3 : commander le transfert chez OVH

Maintenant que votre domaine est déverrouillé et que vous êtes en possession du code de transfert, vous pouvez demander le transfert de votre domaine chez OVH. Pour cela, vous devrez réaliser une commande de transfert depuis [notre site internet](https://www.ovh.com/){.external}.

Saisissez-y votre nom de domaine, puis poursuivez les étapes de la commande. Lorsque vous devrez renseigner le code de transfert, saisissez-le dans la case à côté de votre nom de domaine. Si vous n'êtes pas en possession du code de transfert, une case `Préciser le code d'authentification ultérieurement`{.action} peut être cochée. Dans la mesure du possible, nous vous conseillons de vous assurer au préalable que vous serez en mesure de récupérer ce code avant de poursuivre le processus. 

Les différentes étapes vous permettent de compléter votre commande avec une [offre d'hébergement web](https://www.ovh.com/fr/hebergement-web/){.external}, d'autres solutions OVH, ou de choisir des serveurs DNS. Ces étapes peuvent être intéressantes si le transfert de votre domaine s'inscrit dans un projet de migration de vos services (site et e-mails) chez OVH. Le guide intitulé [Migrer son site et ses e-mails vers OVH](https://docs.ovh.com/fr/hosting/migrer-mon-site-chez-ovh/){.external} vous offrira une vue d'ensemble des démarches à réaliser dans ce cas.

> [!warning]
>
> Nous vous invitons à être particulièrement vigilant sur le choix de vos serveurs DNS lors de la commande. Si votre nom de domaine est utilisé pour un site internet ou des e-mails, vous devrez vous assurer de renseigner les serveurs DNS actuellement utilisés par votre nom de domaine.
>

Une fois le bon de commande affiché, payez ce dernier afin que le transfert du domaine puisse débuter. À la réception de votre paiement, la procédure de transfert sera initiée. Selon l'extension de votre nom de domaine générique, un suivi concernant la procédure de transfert peut être disponible dans votre [espace client OVH](https://www.ovh.com/auth/?action=gotomanager){.external}.

### Étape 4 : confirmer le transfert

Maintenant que le transfert est initié, deux étapes de validation sont nécessaires afin que ce dernier arrive à son terme.

#### Première étape de validation

Cette dernière commence dès le début du transfert et peut durer cinq jours maximum. Deux demandes de validation sont envoyées afin de valider cette première étape.

|Qui reçoit une demande de validation ?|Où la demande de validation est-elle envoyée ?|
|---|---|
|Le propriétaire du domaine|À l'adresse e-mail du propriétaire renseignée dans la base *Whois*|
|L'administrateur désigné lors de la commande chez OVH|À l'adresse e-mail renseignée sur le profil de l'administrateur chez OVH|

Plusieurs scénarios de réponses sont possibles.

|Scénarios de réponses possibles|Suite des événements|
|---|---|
|Validation du propriétaire et de l'administrateur|Passage à la deuxième étape de validation sous 24 heures maximum|
|Une seule validation (qu'importe le contact) et une demande sans réponse|Passage à la deuxième étape de validation au terme du délai de cinq jours|
|Aucune réponse aux deux demandes de validation|Le transfert est abandonné au terme du délai de cinq jours|
|Un refus (qu'importe le contact)|Le transfert est abandonné dès la notification de refus par l'un des contacts|

La validation des deux parties s'effectue depuis une interface OVH. Un lien menant vers cette dernière est présent dans les e-mails envoyés aux deux parties, interface sur laquelle ils devront confirmer ou modifier les informations affichées.

![domain](images/domaintransfer_gTLD_validation.png){.thumbnail}

En cas d'abandon du transfert, assurez-vous que les différentes parties sont en accord avec ce dernier et que leurs adresses e-mail sont à jour. Le transfert pourra être relancé ultérieurement depuis l'[espace client OVH](https://www.ovh.com/auth/?action=gotomanager){.external} dans la section `Domaines`{.action}, puis `Opérations en cours`{.action}.

#### Deuxième étape de validation

Une fois la seconde étape initiée, le bureau d'enregistrement actuel (qui n'est pas encore OVH) reçoit une demande de validation. Plusieurs scénarios de réponses sont possibles.

|Scénarios de réponses possibles|Suite des événements|
|---|---|
|Validation du bureau d'enregistrement actuel|Le transfert est réalisé sous 24 heures maximum|
|Aucune réponse du bureau d'enregistrement actuel|Le transfert est réalisé au terme délai de cinq jours|
|Refus du bureau d'enregistrement actuel|Le transfert est abandonné dès la notification du refus|

En cas de refus du bureau d'enregistrement actuel, nous vous invitons à prendre contact avec ce dernier pour connaître les motivations de ce refus. Le transfert pourra ensuite être relancé depuis l'[espace client OVH](https://www.ovh.com/auth/?action=gotomanager){.external} dans la section `Domaines`{.action}, puis `Opérations en cours`{.action}.

### Étape 5 : gérer votre domaine

Une fois la procédure de transfert terminée, le domaine est maintenant administré depuis votre [espace client OVH](https://www.ovh.com/auth/?action=gotomanager){.external}.

À titre d'information, pour tout transfert payant de domaine générique, nous vous créditons à sa date d'expiration un an supplémentaire.
